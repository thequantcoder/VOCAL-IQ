## Tenancy model (critical)

- Every tenant-owned row has `tenantId` (UUID, indexed, FK).
- Tenants form a hierarchy: **Platform → Reseller → Customer**. A reseller is a tenant with `type=RESELLER` that owns child tenants (`parentTenantId`).
- **Two layers of enforcement:**
  1. **Postgres RLS** — policies keyed on a `app.current_tenant` setting (set per request/connection). Every tenant table has RLS enabled. Migrations in `packages/db/rls/`.
  2. **Application guards** — a NestJS `TenantGuard` resolves the tenant from the auth context and injects it; a `@CurrentTenant()` decorator provides it; repositories always filter by it. The voice service sets the tenant on its DB session per call.
- Reseller isolation: a reseller can read/write only its own subtree. Enforced in both RLS (subtree check) and guards.

## The live call flow (sequence)

1. **Trigger** — campaign job (workers) or API call or inbound webhook (Twilio/SIP) hits `apps/api`, which validates quota, consent, DNC, then asks `apps/voice` to start a session (passing tenant, agent config, lead context).
2. **Connect** — `voice` requests a call leg via the Telephony provider (router) or opens a LiveKit WebRTC room for web calls; bridges media into the Pipecat pipeline.
3. **Loop** — STT (stream) → LLM (stream, with tools/RAG/Squads) → TTS (stream) → caller; barge-in interrupts cleanly; turn-taking model decides endpointing.
4. **Act** — tool nodes call integrations/webhooks; transfers hand off to humans (Agent Desk) or other agents (Squads, shared context bus).
5. **Persist** — audio → R2; transcript + events + sentiment + **per-component cost** → Postgres (Timescale for metrics) in real time.
6. **Close** — disposition tagged; post-call automations fire (SMS/WhatsApp/email/CRM/webhook); summary + keywords generated by a worker.
7. **Reconcile** — usage metered → attributed to tenant → reseller margin computed → pushed to Stripe (metered) / wallet drawn down.

## Provider Router contract (shared TS ↔ Python)

- Input: `RouteRequest { tenantId, agentId, capability, language?, costCeiling?, latencyTarget?, byok? }`.
- Output: a concrete provider client + a `UsageMeter` callback that emits `UsageRecord { provider, capability, units, costUsd, byok }`.
- Selection order: tenant override → plan policy → cost/latency optimisation → availability fallback chain.
- **Every** provider call must go through the router and emit a usage record. No exceptions.

## Realtime transport

- **Signalling + live UI updates:** Socket.IO (rooms namespaced per tenant + per call).
- **Audio media:** LiveKit (WebRTC) for web; SIP/PSTN bridged via Twilio/Telnyx/SIP trunk into LiveKit or directly into Pipecat.
- **Events from voice → api/clients:** transcript partials, node-active, tool-call, sentiment, disposition.

## Security architecture

- Provider keys (platform + tenant BYOK) stored **encrypted** in DB via envelope encryption (KMS data key); decrypted only in memory at call time. Never logged.
- All webhooks (Twilio, Stripe, SIP, WhatsApp) verified by signature + timestamp + replay protection (nonce/dedupe in Redis).
- RBAC enforced in `api` guards on every mutation; roles per `DATA-MODEL.md`.
- Rate limiting + abuse controls (per-tenant call caps, anomaly detection, kill-switch) on calling paths.
- Audit log table records every privileged action (super-admin + reseller-admin).

## Cross-cutting engineering standards (apply throughout; assigned to days)

These are easy to forget — they're owned by the days noted, but every relevant day should respect them:

- **Infra cost monitoring + budget alerts** (distinct from per-call cost attribution): track *infrastructure* + aggregate provider spend; budget alerts to super-admin when daily/monthly spend crosses thresholds; per-tenant spend anomaly alerts. Owned by **Day 41 (analytics)** + **Day 70 (abuse/anomaly)**; cloud-billing alerts set in **Day 62/66 (infra/launch)**.
- **React error boundaries + resilience** (no white-screens; socket-drop recovery): app shell + per-route boundaries with Sentry reporting. Owned by **Day 14** (app shell) and applied on every UI day (`DESIGN-SYSTEM.md §7 Resilience`).
- **Cookie consent + privacy/legal**: consent banner for analytics (PostHog), privacy policy + ToS pages, cookie controls — region-aware. Owned by **Day 60 (compliance)** + surfaced on the **Day 95 (landing)**.
- **Container security**: image scanning (Trivy/Grype) in CI before push; minimal base images; non-root containers; pinned digests. Owned by **Day 1 (CI)** + **Day 64 (security hardening)**.
- **Dependency & secret scanning**: Dependabot/Renovate + gitleaks + SAST in CI. Owned by **Day 1** + **Day 64**.
- **DB connection pooling** (PgBouncer or pooled client) sized for the voice service's concurrency. Owned by **Day 4** + revisited **Day 62/63**.
- **Session replay + product analytics**: PostHog session replay (privacy-masked) + funnels. Owned by **Day 1 (init)** + **Day 50 (onboarding analytics)**.
- **Status page + changelog**: public status page + a customer-facing changelog/release notes. Owned by **Day 66 (launch readiness)**.
- **In-app help**: knowledge base + the support ticketing from **Day 49**; optional live-chat widget (defer post-launch).
- **Email deliverability**: SPF/DKIM/DMARC on sending domains. Owned by **Day 15 (transactional)** + **Day 72 (marketing email)**.
